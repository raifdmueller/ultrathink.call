== Risks and Technical Debt

Risks and technical debt are kept separate. Risks are ordered by priority; each
carries probability, impact, a derived priority, and a mitigation that references
a Chapter 8 mitigation or a Chapter 10 scenario. Each debt item names the
Chapter 5 building block it burdens.

Priority is derived from probability × impact, then adjusted by exposure on the
primary path: a threat on the call's own signaling/delivery path outranks one
that first requires compromising a third party. This rule makes the High/Med
split reproducible (e.g. it separates R-7 from R-10 at equal Low/High inputs).
