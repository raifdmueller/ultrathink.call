== Context and Scope

The system is one ultrathink.call instance running in a participant's browser.
Everything outside that browser tab is an external system: the other
participant's instance, the sovereign STUN server, the out-of-band channel that
carries the invite, and the static host that delivers the app.

=== Business Context

[cols="1,2,2",options="header"]
|===
| Neighbour | Input to the system | Output from the system
| Host / Guest (user) | Camera/mic, device choice, paste of an invite/answer | Local + remote video, the invite/answer capability-URL
| Other participant | The returned answer (or invite) token | The offer (or answer) token
| Out-of-band channel (email/RCS) | — | The `mailto:` draft carrying the capability-URL
| Sovereign STUN server | ICE candidates (public reflexive address) | STUN binding requests
| Static host (GitHub Pages) | — | Delivers the static page over HTTPS
|===

=== Scope boundary

In scope: media capture, the WebRTC peer connection, the capability-URL token
codec and validation, device selection, screen sharing, and the `mailto:` invite.
Out of scope (external): the mail transport, the STUN operator, the static host,
and the remote peer's runtime. There is deliberately no backend in scope (TC-1).
