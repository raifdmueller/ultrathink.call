== Building Block View

=== Level 1 — Whitebox ultrathink.call

The static app decomposes into six building blocks. The external systems are the
same set as Chapter 3. Since EPIC 1 (issue #7, done) the blocks live in separate
ES modules under `src/app/` — `codec.js`, `peer.js`, `signaling.js`, `main.js` —
loaded natively without a build; the source-location column names each module.

=== Building blocks

[cols="1,2,2,2",options="header"]
|===
| Block | Responsibility | Interface | Source location
| BB-1 UI Shell | Present controls, video tiles, status; wire DOM events to the other blocks. | DOM events; `status()` text. | `index.html` (markup); `src/app/main.js` (event wiring)
| BB-2 Media Manager | Capture camera/mic, enumerate and switch devices, screen share, restore camera. | `getUserMedia`, `enumerateDevices`, `replaceTrack`. | `src/app/main.js`: `populateDevices`, `switchDevices`, `restoreCamera` + `#startCam`/`#shareScreen` handlers

| BB-4 Signaling Transport (port) | Produce and consume capability tokens; current adapter = capability-URL + `mailto:` + clipboard. | BB-6 Validation | Enforce decode-time rules VR-1..6,8,9 and the post-decode guard GD-1. | validation inside `decodePayload`; guard in handlers. | `src/app/codec.js` (`decodePayload`); `src/app/main.js` (`if (session)` guards)

=== Level 2 — Signaling Transport (port) whitebox

BB-4 is the architecture's main seam (QG-4). It exposes a transport-agnostic
contract — "publish an offer / await an answer", and the reverse — over which two
adapters live: the implemented *Manual adapter* (capability-URL token + `mailto:`
+ clipboard) and the planned *Broker adapter* (R2: automatic exchange over a
sovereign non-US broker, issue #7). Swapping adapters must not touch BB-3 or BB-5.
