== Quality Requirements

Chapter 1.2 names the top goals that drive decisions; this chapter covers all
eight ISO/IEC 25010 characteristics. Each entry is marked as concretising a
Chapter 1.2 goal (QG-*) or as a derived requirement. Scenarios use the six-part
form (Source, Stimulus, Artifact, Environment, Response, Response Measure) with a
literal measure, so each is testable.

=== Quality tree

[cols="1,2,1",options="header"]
|===
| ISO 25010 characteristic | In this system | Concretises / derived
| Security | Sovereignty, encryption, input validation | QG-1 (top goal)
| Reliability | Degrade under load, not drop; bounded waits | QG-3 (top goal)
| Maintainability | Ports + adapters, swappable dependencies | QG-4 (top goal)
| Functional Suitability | 2→4 call, screen share, device selection | derived
| Usability | Join < 60 s, no install, no login | derived (supports QG-2)
| Performance Efficiency | Token fits a link; client-side codec | derived
| Compatibility | Modern-browser WebRTC, no native app | derived
| Portability | Static page runs on any static host | derived (supports QG-2)
|===

NOTE: Operability/zero-ops (QG-2) is realized structurally (no server exists,
Chapter 7) rather than as a runtime scenario; QS-4 records its measure.
